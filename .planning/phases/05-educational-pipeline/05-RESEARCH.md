## Summary

Phase 5 adds an "Educational" content mode to the v2 Web UI pipeline. The full change set is 10 discrete code gaps spread across 8 files, plus one new agent (`src/agents/pdf_guide/agent.py`) and frontend work in the separate `techwithhareen-web` repo. The gap analysis in `docs/brainstorms/2026-03-31-v5-deep-dive.md` is accurate and comprehensive — every gap maps precisely to existing code read during research.

The biggest new piece is the **PDF Guide Agent**, which requires ReportLab (not yet in `requirements.txt`), a `BytesIO`-based render pipeline, and GCS upload via `blob.upload_from_string()` — a different upload path from the existing `_upload_sync()` function that uses `upload_from_filename`. The carousel renderer modification is surgical: one new `_slide_learn_preview()` builder function and a `content_type` parameter threaded into `render_carousel()`. Everything else is small additions to existing agents.

**Important dependency constraint:** Phase 2 (Content Classification) adds `content_type` to Story and runs before carousel creation. Phase 5 depends on CLASS-01 through CLASS-03 being complete so `content_type` exists on Story. However, Phase 5 uses a simplified 2-value type (`"news"` / `"educational"`) rather than the full 3-value Phase 2 classifier (`news_and_announcements` / `tool_and_product` / `educational`). The Story field introduced in Phase 5 must be compatible with Phase 2's later expansion — use `Optional[str]` with a string value rather than a locked `Literal` until Phase 2 is done.

**Primary recommendation:** Build in this order — (1) Story + ResearchRequest field additions, (2) PDFGuideAgent skeleton with ReportLab, (3) carousel renderer educational branch, (4) pipeline branch in `_run_research_pipeline`, (5) Caption Writer educational voice, (6) Firestore fields, (7) frontend. Each step is independently testable.

### Pattern 1: ReportLab Canvas with BytesIO (no temp file)

**What:** Generate PDF bytes in memory, upload directly to GCS without writing to disk.
**When to use:** All PDF generation in PDFGuideAgent.

```python
# Source: Django docs / ReportLab official pattern (verified via web search)
import io
from reportlab.pdfgen import canvas
from reportlab.pdfbase import pdfmetrics
from reportlab.pdfbase.ttfonts import TTFont

# Register TTF fonts once (module level)
pdfmetrics.registerFont(TTFont("Anton", str(FONTS_DIR / "Anton-Regular.ttf")))
pdfmetrics.registerFont(TTFont("Inter", str(FONTS_DIR / "Inter-Regular.ttf")))
pdfmetrics.registerFont(TTFont("InterSB", str(FONTS_DIR / "Inter-SemiBold.ttf")))

def render_pdf(guide_content: dict) -> bytes:
    buffer = io.BytesIO()
    c = canvas.Canvas(buffer, pagesize=(612, 792))  # US Letter
    # ... draw content ...
    c.showPage()
    c.save()
    return buffer.getvalue()
```

### Pattern 2: GCS Upload from Bytes (no temp file)

**What:** Upload PDF bytes directly from memory to GCS.
**When to use:** PDFGuideAgent after `render_pdf()`.

```python
# Source: GCS official docs sample (storage-file-upload-from-memory)
# Different from existing _upload_sync() which uses upload_from_filename
def _upload_pdf_sync(pdf_bytes: bytes, gcs_object_name: str) -> str:
    client = _get_gcs_client()
    bucket = client.bucket(GCS_BUCKET)
    blob = bucket.blob(gcs_object_name)
    blob.upload_from_string(pdf_bytes, content_type="application/pdf")
    return f"{GCS_BASE_URL}/{gcs_object_name}"
```

### Pattern 4: Educational Pipeline Branch

**What:** `_run_research_pipeline` branches on `content_type` after research step.
**When to use:** Educational flow skips ContentValidator and runs PDFGuideAgent before CaptionWriter.

```python

async def _run_research_pipeline(job_id: str, topic: str, content_type: str = "news") -> None:
    # Step 1 — Research (always)
    if content_type == "educational":

    else:

            dm_keyword = None
            pdf_url = None

        caption = await _caption_writer.run(story, carousel, dm_keyword=dm_keyword)
        # ...
        post_id = await db.create_post(
            story, slides, caption_text, source="web_ui",
            render_data=render_data,
            pdf_url=pdf_url,
            dm_keyword=dm_keyword,
            content_type=content_type,
```

### Anti-Patterns to Avoid

- **Using `Literal["news", "educational"]` on Story.content_type:** Phase 2 will expand this to 3 values. Use `Optional[str]` to avoid a breaking change.
- **Running ContentValidator for educational posts:** It does 2 LLM calls that return trivially-passing results for a pre-authored guide topic. Skip it entirely — the guard belongs in `_run_research_pipeline`.
- **Writing PDF to /tmp before uploading:** Use `BytesIO` + `upload_from_string()` — keeps the Cloud Run container clean and avoids disk I/O.
- **Using Platypus SimpleDocTemplate for the PDF:** Platypus is designed for flowing documents (articles, reports). The UncoverAI brand guide has precise positioning requirements. Use the lower-level `canvas.Canvas` API for full layout control.
- **Embedding Anton font without registering it:** ReportLab falls back to Courier silently if `pdfmetrics.registerFont()` is not called before the canvas is created. Register at module level.
- **Calling `ResearchOrchestrator.run()` for educational topics:** The news synthesis prompt generates 5 angles and fills `hook_stat_value` — both wrong for educational. Always route through `run_educational()`.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| PDF layout and rendering | Custom PNG-to-PDF stitching or HTML templates | ReportLab canvas API | Font embedding, page sizing, vector text — solved and well-tested |
| Font embedding in PDF | Custom font subsetting | `pdfmetrics.registerFont(TTFont(...))` | ReportLab handles subset embedding automatically |
| In-memory PDF output | temp file on /tmp | `io.BytesIO` as canvas target | No disk I/O, cleaner Cloud Run container |
| GCS upload from memory | `upload_from_filename` with BytesIO | `blob.upload_from_string(pdf_bytes, content_type="application/pdf")` | Designed for byte strings / in-memory data |

**Key insight:** ReportLab is the only pure-Python PDF library with solid TTF embedding that runs without system dependencies. Everything else either requires Chromium (WeasyPrint/pdfkit) or has worse font support (fpdf2). The BytesIO pattern is idiomatic ReportLab — used in Django docs as the canonical example.

## Common Pitfalls

### Pitfall 1: content_type not threaded through `_run_research_pipeline`
**What goes wrong:** `_run_research_pipeline` receives `content_type` from `ResearchRequest` but the caller `start_research()` only passes `topic`. The educational branch never fires.
**Why it happens:** `asyncio.create_task(_run_research_pipeline(job_id, topic))` — `content_type` is dropped.
**How to avoid:** Change call site in `start_research()` to pass `content_type`: `asyncio.create_task(_run_research_pipeline(job_id, topic, content_type=body.content_type))`.
**Warning signs:** Educational posts go through ContentValidator and produce news-style carousel.

### Pitfall 2: ReportLab falls back to Courier font silently
**What goes wrong:** Anton and Inter fonts render as Courier in the PDF — brand completely broken.
**Why it happens:** `pdfmetrics.registerFont()` not called before `canvas.Canvas()` is instantiated, or font path doesn't exist on Cloud Run container.
**How to avoid:** Register fonts at module level in `agent.py`, same pattern as `_font()` in `carousel_renderer.py`. Check `Path(font_path).exists()` before registering and fall back gracefully.
**Warning signs:** PDF renders on local machine but not on Cloud Run — font paths differ.

### Pitfall 3: `buffer.getvalue()` called before `canvas.save()`
**What goes wrong:** `BytesIO.getvalue()` returns empty bytes or incomplete PDF — GCS upload corrupted.
**Why it happens:** PDF bytes are only flushed to the buffer when `canvas.save()` is called. If `getvalue()` is called before `save()`, the buffer is empty.
**How to avoid:** Always call `c.save()` before `buffer.getvalue()`. Canonical order: `c.showPage()` → `c.save()` → `buffer.getvalue()`.

### Pitfall 4: `_run_research_pipeline` processes multiple stories for educational
**What goes wrong:** If `_synthesise_educational()` somehow returns >1 story, the pipeline runs PDF generation multiple times — duplicate PDFs.
**Why it happens:** The educational synthesis prompt says "return exactly 1 story" but LLMs occasionally ignore this.
**How to avoid:** Hard-cap in `_run_research_pipeline` educational branch: `passing_stories = stories[:1]`.

### Pitfall 5: Story `content_type` field missing from `to_dict()`
**What goes wrong:** Firestore stores the Story without `content_type` — the web UI can't render the DM keyword badge correctly because `post.story.content_type` is null.
**Why it happens:** `Story.to_dict()` must be updated to include `content_type` when it's added to the dataclass.
**Warning signs:** DM keyword badge appears in Firestore doc root but not in `story` sub-document.

### Pitfall 6: `dm_keyword` not passed to CaptionWriter
**What goes wrong:** Educational caption uses the default "Send this to someone" CTA instead of "DM me CLAUDE for the full guide".
**Why it happens:** PDFGuideAgent runs and returns `dm_keyword`, but the result is not wired into the `CaptionWriterAgent.run()` call in `_run_research_pipeline`.
**How to avoid:** In the educational pipeline branch, capture `pdf_result.dm_keyword` and pass it explicitly to `CaptionWriterAgent.run(story, carousel, dm_keyword=dm_keyword)`.

### ReportLab: Register TTF fonts and write to BytesIO
```python
# Source: ReportLab ch3_fonts docs + canonical Django outputting-pdf example
import io
from pathlib import Path
from reportlab.pdfgen import canvas
from reportlab.pdfbase import pdfmetrics
from reportlab.pdfbase.ttfonts import TTFont

FONTS_DIR = Path(__file__).parent.parent.parent.parent / "assets" / "fonts"

def _register_fonts():
    for name, filename in [
        ("Anton", "Anton-Regular.ttf"),
        ("Inter", "Inter-Regular.ttf"),
        ("InterSB", "Inter-SemiBold.ttf"),
    ]:
        path = FONTS_DIR / filename
        if path.exists():
            pdfmetrics.registerFont(TTFont(name, str(path)))

_register_fonts()  # module-level call

def render_pdf_to_bytes(content: dict) -> bytes:
    buffer = io.BytesIO()
    c = canvas.Canvas(buffer, pagesize=(612, 792))
    # ... draw content ...
    c.showPage()
    c.save()           # MUST call save() before getvalue()
    return buffer.getvalue()
```

## Open Questions

1. **PDF page dimensions — A4 vs US Letter vs Instagram aspect ratio?**
   - What we know: ReportLab supports arbitrary page sizes; `(612, 792)` is US Letter in points; A4 is `(595, 842)`
   - What's unclear: The BRD/PRD don't specify. A "mini-guide" could be single-page or multi-page.
   - Recommendation: Use A4 landscape `(842, 595)` or US Letter `(612, 792)` for print-readability. Single-page first draft — Cover + Intro + Steps inline. Add pages if content overflows.

2. **How many lesson steps per educational post?**
   - What we know: `key_stats` currently holds 8-12 items (news stats); educational guide needs 5-8 lesson steps
   - What's unclear: The synthesis prompt for educational needs a clear target number
   - Recommendation: Prompt for 5-7 lesson steps in `_synthesise_educational()`. This fills 2 content slides (4+3) cleanly.

3. **PDF font fallback on Cloud Run if Anton.ttf not in assets/?**
   - What we know: The carousel renderer already has a `_SYSTEM_BOLD` fallback list for Anton
   - What's unclear: Whether the same `assets/fonts/` directory is baked into the Docker image
   - Recommendation: Check the Dockerfile for `COPY assets/ assets/` — if it's present for the carousel renderer, it's present for the PDF agent too.

4. **Should `_run_research_pipeline` be refactored or just branched?**
   - What we know: The function is ~80 lines; adding a branch adds ~30 lines
   - What's unclear: Whether this is clean enough or warrants extraction into separate helper functions
   - Recommendation: Simple conditional branch in the existing function is fine for Phase 5. Extraction can happen if a Phase 6 adds a third content mode.
