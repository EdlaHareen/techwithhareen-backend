# v5 — Educational Content Pipeline

## What We're Building

A new `educational` content type in the v2 pipeline. When Hareen types a learning topic in the Web UI (e.g. "how to use Claude for work"), the pipeline produces:
1. An educational carousel (step-by-step, actionable slides)
2. A PDF mini-guide — auto-generated, uploaded to GCS, previewed in Web UI
3. An auto-generated DM keyword (e.g. "CLAUDE") tied to the topic

CTA on last slide + caption: "DM me [KEYWORD] for the full guide"
DM delivery: manual first (Hareen sends GCS link) → ManyChat automation later

## Why This Approach

Current content is 100% news-driven (Pillar 1). Educational content (Pillar 2) drives saves, DMs, and follows — the signals Instagram's algorithm rewards most. Mini-guide as lead magnet also builds toward future monetization (paid courses, newsletters).

Approach B chosen over:
- A (carousel only) — misses the lead magnet / DM growth opportunity
- C (manual PDF) — doesn't scale

## Key Decisions

- **Trigger**: v2 Web UI — Hareen selects "Educational" content type when typing topic
- **Carousel structure**: Step-by-step lesson slides (not news bullets) — numbered, actionable takeaways per slide
- **Caption voice**: New `educational` mode added to Caption Writer Agent
- **PDF Guide Agent**: New agent — ReportLab or WeasyPrint, GCS upload, public URL returned
- **Keyword**: Auto-generated by LLM from topic (e.g. "how to use Claude for work" → "CLAUDE")
- **PDF preview**: Shown in Web UI approval queue before Hareen approves
- **DM delivery**: Manual at first → ManyChat keyword automation later
- **Research**: Same Exa+Tavily+Serper stack, queries tuned for tutorials/how-tos not news

## Content Pillars

| Pillar 1 (existing) | Pillar 2 (v5) |
|---|---|
| News & updates | Education & skills |
| Reactive to news cycle | Evergreen, saves forever |
| Drives views | Drives saves, DMs, follows |
