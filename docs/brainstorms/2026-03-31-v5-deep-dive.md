# v5 Deep Dive — Gap Analysis & Revised Plan

Reviewed against actual codebase. Every gap below is a real code-level missing piece.

---

## What the brainstorm got right

- `educational` content type as a mode switch — correct
- Same Exa+Tavily+Serper stack, queries tuned differently — correct
- PDF Guide Agent as new agent — correct
- Caption Writer gets new `educational` voice — correct
- PDF preview in Web UI — correct
- Auto-generated DM keyword — correct

---

## Gaps the brainstorm missed

### Gap 1 — `Story` dataclass has no `content_type` field

**File:** `src/utils/story.py`

`Story` currently has no way to carry `content_type`. Every agent downstream
(carousel renderer, caption writer, PDF agent) needs to know if it's working
on news or educational content. Without this field, you can't branch behaviour
anywhere in the pipeline.

**Fix:** Add `content_type: Literal["news", "educational"] = "news"` to `Story`
and include it in `to_dict()`.

---

### Gap 2 — `ResearchRequest` API model has no `content_type`

**File:** `src/api/routes_v2.py` line 44

```python
class ResearchRequest(BaseModel):
    topic: str  # ← content_type missing
```

The Web UI has no way to tell the backend "this is educational". The backend
can't branch the pipeline without it.

**Fix:** Add `content_type: Literal["news", "educational"] = "news"` to
`ResearchRequest`. Pass it into `_run_research_pipeline`.

---

### Gap 3 — Research Orchestrator synthesis prompt is news-only

**File:** `src/agents/research_orchestrator/orchestrator.py` line 128

The current `_synthesise` prompt asks Claude to find "5 best distinct story
angles" and fill in `hook_stat_value` / `hook_stat_label` (shock stats for
news). For educational posts this is wrong in two ways:

1. **Volume**: Educational content = 1 comprehensive post per topic, not 5 news
   angles. The orchestrator must return exactly 1 Story for educational.
2. **Content shape**: `key_stats` for educational should be lesson steps
   ("STEP 1: OPEN A NEW CONVERSATION\nStart with a clear task description"),
   not news facts. `hook_stat_value` / `hook_stat_label` are irrelevant.
3. **Image query**: For "how to use Claude for work", the `image_query` must be
   `"Claude AI logo"` or `"Claude interface screenshot"` — not a news thumbnail.
   The synthesis prompt must generate visual-asset queries, not article queries.

**Fix:** Add a separate `_synthesise_educational(topic, results)` method with
a different prompt. Called when `content_type == "educational"`.

---

### Gap 4 — Carousel renderer has no educational slide mode

**File:** `src/utils/carousel_renderer.py`

Slide 2 is always a "Hook Stat" — large accent number + white label. For
educational posts, `hook_stat_value` will be empty. What renders? Likely a
blank or broken slide.

Educational Slide 2 should instead show:
```
WHAT YOU'LL LEARN
────────────────
[3-4 bullet preview of the steps]
```

The renderer needs a conditional: if `hook_stat_value` is empty AND
`content_type == "educational"`, render a "What You'll Learn" slide instead.

**Fix:** Add `content_type` param to `create_carousel()` and the renderer.
Add `_render_learn_preview_slide()` method.

---

### Gap 5 — `ContentValidator` will waste LLM calls on educational posts

**File:** `src/agents/content_validator/validator.py`

The validator does 3 LLM checks: relevance, freshness, deduplication. For
educational content:
- Relevance check: trivially passes — the one story IS the topic
- Freshness check: no date available — defaults to not-stale (fine but pointless)
- Deduplication: only 1 story → no-op

Running the validator for educational posts burns LLM tokens for zero value.

**Fix:** In `_run_research_pipeline`, skip `ContentValidator` when
`content_type == "educational"`.

---

### Gap 6 — PDF Guide Agent doesn't exist yet

**New file:** `src/agents/pdf_guide/agent.py`

This is the biggest new piece. It needs to:

1. **Generate guide content** (LLM call) — given the story's `key_stats`
   (lesson steps), produce expanded guide text: intro paragraph, each step
   with a 2-3 sentence explanation, tips section, resource links.
2. **Generate DM keyword** (same LLM call) — e.g. "how to use Claude for
   work" → `"CLAUDE"`. Single uppercase word, max 8 chars.
3. **Render PDF** — ReportLab (pure Python, no HTML/CSS dependency).
   Brand it: dark navy BG (#1A1A2E), periwinkle accent (#8075FF), same
   Anton/Inter fonts.
   Sections: Cover → Intro → Steps → Tips → Resources
4. **Upload to GCS** — same bucket as carousels
   (`techwithhareen-carousel-assets`), path: `guides/{post_id}.pdf`
5. **Return** `pdf_url: str` + `dm_keyword: str`

ReportLab chosen over WeasyPrint because it has no system-level HTML/CSS
rendering dependency — cleaner on Cloud Run.

---

### Gap 7 — Caption Writer has no `educational` story type

**File:** `src/agents/caption_writer/agent.py`

Current story types: `tool_feature`, `funding_acquisition`, `research_finding`,
`general_news`. The educational voice mode is different:

```
EDUCATIONAL
→ You're the teacher. Make it feel like a quick cheat code, not a lecture.
→ Hook: "Here's how to [do X] — the part most people skip."
→ Body: What they'll get from the guide. Make it concrete and actionable.
→ CTA: "DM me [KEYWORD] for the full guide 📩"
→ Hashtags: same 3-5 rule, focus on skill/tool tags
```

The `dm_keyword` must be injected into the CTA. This means the Caption Writer
needs access to the keyword — which comes from the PDF Guide Agent. So in the
pipeline, **PDF Guide Agent must run before Caption Writer** for educational posts.

**Fix:** Add `educational` story type + voice to `_PERSONA_INSTRUCTION`. Add
`dm_keyword: Optional[str]` param to `CaptionWriterAgent.run()`. Inject into
CTA when present.

---

### Gap 8 — Firestore `create_post` doesn't store `pdf_url`, `dm_keyword`, `content_type`

**File:** `src/utils/firestore_client.py`

`db.create_post()` stores slides, caption, story, render_data. It has no
`pdf_url`, `dm_keyword`, or `content_type` fields. The Web UI can't show PDF
preview without `pdf_url`. The keyword disappears after pipeline runs.

**Fix:** Add optional `pdf_url`, `dm_keyword`, `content_type` params to
`create_post()`. Include in the Firestore document.

---

### Gap 9 — `_run_research_pipeline` doesn't have an educational branch

**File:** `src/api/routes_v2.py` line 284

The background pipeline function needs two branches:
- News (`content_type == "news"`): current flow (validator → pipeline → store)
- Educational (`content_type == "educational"`):
  1. Skip ContentValidator
  2. Run PostCreatorAgent (educational carousel)
  3. Run PDFGuideAgent → get `pdf_url` + `dm_keyword`
  4. Run CaptionWriterAgent with `dm_keyword`
  5. Run PostAnalyzerAgent
  6. Store with `pdf_url`, `dm_keyword`, `content_type`

**Fix:** Branch in `_run_research_pipeline` after research step. No structural
changes needed — just a conditional path.

---

### Gap 10 — Web UI needs 3 changes (separate repo: `techwithhareen-web`)

1. **Content type toggle** on the research trigger form — "News" / "Educational"
   pill selector. Sends `content_type` in the POST body.
2. **PDF preview panel** in the approval queue PostCard — shown only when
   `post.pdf_url` is set. Can be a simple `<a>` link to open the PDF, or an
   embedded `<iframe>`.
3. **DM keyword badge** on the PostCard — "DM keyword: CLAUDE" shown when
   `post.dm_keyword` is set.

These are frontend changes in `techwithhareen-web` repo.

## Is the scope right? Are we doing too much?

**No, the scope is right.** Every piece is necessary for the feature to work
end-to-end. Nothing is speculative.

The PDF Guide Agent is the most work (~150 lines: LLM call + ReportLab + GCS).
Everything else is small modifications to existing agents.

**Rough breakdown:**
| Change | Effort |
|---|---|
| `Story` dataclass — add `content_type` | 5 min |
| `ResearchRequest` — add `content_type` | 5 min |
| Research Orchestrator — educational synthesis prompt | 30 min |
| Carousel renderer — educational slide 2 | 30 min |
| Skip ContentValidator for educational | 5 min |
| PDF Guide Agent (new) | 2-3 hours |
| Caption Writer — educational voice + keyword | 30 min |
| Firestore — new fields | 15 min |
| `_run_research_pipeline` — educational branch | 30 min |
| Web UI — toggle + PDF preview + keyword badge | 1 hour |

Total: ~6 hours of focused work.

```

  ↓

  ↓
ResearchOrchestrator._synthesise_educational()
  - Exa/Tavily: tutorial/how-to content
  - Serper: tool visuals (logo, screenshot)
  - Returns exactly 1 Story with lesson steps in key_stats
  ↓
[ContentValidator SKIPPED]
  ↓
PostCreatorAgent
  - image_query: "Claude AI logo" (visual-asset query from story)
  - Carousel with educational Slide 2 ("WHAT YOU'LL LEARN")
  ↓
PDFGuideAgent (NEW — runs before CaptionWriter)
  - Expand lesson steps → full guide text
  - Generate dm_keyword ("CLAUDE")
  - Render PDF (ReportLab + brand)
  - Upload to GCS → pdf_url
  ↓
CaptionWriterAgent
  - educational voice mode
  - CTA: "DM me CLAUDE for the full guide 📩"
  ↓
PostAnalyzerAgent
  ↓
Firestore /posts (with pdf_url, dm_keyword, content_type)
  ↓
Web UI approval queue
  - Carousel preview (existing)
  - PDF preview link/iframe (new)
  - DM keyword badge (new)
```
